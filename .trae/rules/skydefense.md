# PROJECT RULES — SKY DEFENSE

## 1. FRAMEWORK & DEPENDENCIES

Flutter (latest stable), Dart, Flame, Riverpod, GoRouter, Hive.
Use only stable, production-ready libraries.

## 2. TESTING

Use Flutter test framework.
Code must be modular, decoupled, and testable (Clean Architecture).

## 3. PROHIBITED

No Firebase, APIs, backend, cloud services, or sandbox environments.
All code must run locally.
No sandbox environments.
## REQUIREMENTS

Offline-first app.
Store all data locally (Hive).
No hardcoded UI text (use ES/EN localization).
Support real-time language switching.
Use Repository Pattern and Clean Architecture.
Code must be scalable and production-ready.
